# AIDM Documentation Index

This is the navigation entry point for maintained project documentation outside
README files. The code, configuration, tests, migrations, scripts, and
deployment workflows remain the source of truth when a document disagrees with
the implementation.

Status labels used below:

- **Active**: current guidance that should be updated with behavior changes.
- **Template**: a reusable form or checklist; blank fields are not project
  status.
- **Executable contract**: stored under `docs/` but loaded or exercised by the
  application; changes can alter product behavior.
- **Point-in-time**: a dated report that preserves historical evidence.
- **Historical**: retained for context only; verify every claim before reuse.

## Deliberately Excluded

All README variants are outside this documentation set. Generated release
evidence, dependency trees, vendored content, build output, caches, archives of
runtime artifacts, and third-party files are also excluded from active
documentation maintenance.
